# EnigmaHack_Quadro
решение для хакатона enigma hack: ии-агент классификации писем, извлечения данных и генерации ответов на основе базы данных


## Пользовательский путь (User Journey)

1. **Получение письма**  
   Система автоматически подключается к почтовому ящику техподдержки (Яндекс.Почта) по протоколу IMAP и забирает новые непрочитанные письма.

2. **Обработка AI-агентом**  
   Текст письма передаётся в ML-модуль, который:
   1. классифицирует обращение (например: «проблема с заказом», «оплата», «возврат», «другое»);
   2. извлекает ключевые сущности (номер заказа, имя клиента, дату) с помощью NLP-модели.

3. **Генерация проекта ответа**  
   На основе определённой категории система обращается к базе знаний (таблица `knowledge_base` в PostgreSQL), выбирает подходящий шаблон и подставляет извлечённые данные. Формируется предварительный ответ.

4. **Отображение в веб-таблице**  
   Оператор (пользователь системы) видит в интерфейсе (React-таблицу) все обращения с колонками:
   1. исходный текст письма;
   2. категория;
   3. извлечённые данные;
   4. сгенерированный ответ;
   5. статус (новое, обработано, отправлено).

5. **Редактирование и отправка**  
   Оператор может отредактировать ответ при необходимости и нажать кнопку «Отправить». Система отправляет письмо клиенту через SMTP (Яндекс.Почта) и меняет статус обращения на «отправлено».
